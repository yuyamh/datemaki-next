# ルール

## フォーマッタとESLint

コーディングルールは基本的に以下の設定ルールに従うこと。

- eslint.config.mjs
- prettier.config.mjs

## Next.js

- 基本的にRoute Handlerの形式に従うこと。
- また、実装者が理解しやすいような文法で記述すること。
- 適度に、コメントアウトを入れることで追いやすくすること。
- interfaceはsrc/app/lib/interfaces/以下に寄せること。
- DBへのアクセスロジックは、なるべくroute.tsに寄せること。

## ライブラリ

- UIは自作するのではなく、できる限りshadcn/uiをインポートして利用すること。
- アイコンはLucideを使用すること。

## その他ルール

- 改修においては、コメントアウトはそのまま残しておくこと。
- 改修内容とズレる場合は、適宜コメントアウトも改修して残しておくこと。
